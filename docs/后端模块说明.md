# Goster-IoT 后端模块说明

## 1. 总览

当前后端主链路如下：

```text
go/cli
  -> go/src/config
  -> go/src/persistence
  -> go/src/identity
  -> go/src/core
  -> go/src/web
  -> go/src/iot_gateway

go/src/core
  -> go/src/device_manager
  -> go/src/storage/runtime

go/src/web/v1
  -> go/src/identity
  -> go/src/inter
  -> go/src/device_manager 提供的业务接口
  -> go/src/storage/runtime 提供的查询接口
```

## 2. 模块清单

### 2.1 `go/cli`

**用途**

- 作为后端进程入口。
- 负责加载配置、初始化日志、打开认证存储和运行时存储。
- 负责装配 `identity`、`core`、`web`、`iot_gateway`。
- 提供命令行入口：`serve`、`db init`、`db migrate`。

**依赖模块**

- `go/src/config`
- `go/src/logger`
- `go/src/persistence`
- `go/src/identity`
- `go/src/core`
- `go/src/web`
- `go/src/iot_gateway`
- `go/src/inter`

**提供的接口**

- `Run()`
- `RunWithArgs(ctx context.Context, args []string) error`
- `StartWithContext(ctx context.Context)`

**需要的配置**

- `config.AppConfig`
- 直接使用：
  - `AppConfig.DB`
  - `AppConfig.Web`
  - `AppConfig.API`
  - `AppConfig.Auth`
  - `AppConfig.Captcha`
  - `AppConfig.DeviceManager`
  - `AppConfig.Logger`

### 2.2 `go/src/config`

**用途**

- 定义后端统一配置模型。
- 提供默认值、归一化逻辑和加载入口。
- 作为 `cli`、`web`、`iot_gateway`、`device_manager`、`persistence` 的配置来源。

**依赖模块**

- `go/src/logger`
- `viper`

**提供的接口**

- `AppConfig`
- `DBConfig`
- `WebConfig`
- `APIConfig`
- `AuthConfig`
- `CaptchaConfig`
- `DeviceManagerConfig`
- `Load()`
- `DefaultAppConfig()`
- 各类 `Default*Config()` / `Normalize*Config()`

**需要的配置**

- 本模块本身不消费外部配置，它定义并归一化所有后端配置。

**用途**

- 提供统一日志实现和默认 logger。

**依赖模块**

**提供的接口**

**需要的配置**

  - `Format`

### 2.6 `go/src/identity`

**用途**

- 封装 `Authboss` 认证能力。
- 提供认证服务接口，供 `web/v1` 调用。
- 解析当前请求主体，把用户和租户角色组合成 `RequestPrincipal`。
- 提供认证链路使用的最小存储接口定义。

**依赖模块**

- `go/src/config`
- `go/src/inter`
- `authboss`
- `gorilla/sessions`

**提供的接口**

- 认证服务：
  - `type Service interface`
  - `NewAuthbossService(ab *authboss.Authboss) (Service, error)`
- 认证存储：
  - `type Store interface`
- 主体解析：
  - `type PrincipalResolver interface`
  - `NewTenantPrincipalResolver(dataStore tenantRoleStore) PrincipalResolver`
- Authboss 装配：
  - `SetupAuthboss(store Store) (*authboss.Authboss, error)`
  - `SetupAuthbossWithConfig(store Store, cfg appcfg.AuthConfig) (*authboss.Authboss, error)`
- 用户模型：
  - `AuthUser`
- Session / client state：
  - `SessionStorer`
  - `SessionState`

**需要的配置**

- `config.AuthConfig`
  - `RootURL`
  - `CookieSecure`
  - `GitHubClientID`
  - `GitHubClientSecret`
  - `SessionCookieMaxAgeSeconds`
  - `RememberCookieMaxAgeSeconds`

## 3. 新增代码时的放置规则

### 3.1 新增 HTTP 接口

- 放在 `go/src/web/v1`
- 如果是新版本接口，放在 `go/src/web/v2`

### 3.2 新增认证能力

- 认证流程与 Authboss 适配放在 `go/src/identity`
- 认证数据落库放在 `go/src/storage/identity`

### 3.3 新增设备域业务

- 业务服务放在 `go/src/device_manager`
- 存储实现按业务域放在 `go/src/storage/*`

### 3.4 新增数据库表或字段

- schema 和 migration 放在 `go/db`
- 运行时访问代码放在 `go/src/storage/*`

### 3.5 新增运行时存储后端

- 打开逻辑放在 `go/src/persistence`
- 业务域实现放在 `go/src/storage/*`
